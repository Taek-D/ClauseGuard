## 1. 프로젝트 개요

### 미션

> **계약서 리뷰를 법률 전문가의 전유물에서 모든 비즈니스의 기본 역량으로 전환한다.**

### 비전

2028년까지 SMB/중견기업이 체결하는 B2B 계약의 **30% 이상**이 AI 사전 리뷰를 거치는 세상을 만든다. ClauseGuard는 그 표준 도구가 된다.

### 핵심 가치 제안 (Core Value Proposition)

| 차원 | 현재 상태 (As-Is) | ClauseGuard (To-Be) |
|------|-------------------|---------------------|
| **비용** | 변호사 검토 건당 $500~$5,000 | 월 $99~$499 무제한 리뷰 |
| **속도** | 계약서 1건 리뷰에 3~7일 | **5분 이내** 자동 분석 완료 |
| **접근성** | 법무팀 또는 외부 로펌 의존 | 비법률 직군도 즉시 사용 가능 |
| **일관성** | 검토자마다 기준 상이 | 산업별 표준 리스크 프레임워크 기반 |
| **커버리지** | 핵심 계약만 선별 검토 | **모든 계약** 전수 리뷰 가능 |

### 원라이너

**"업로드하면 5분 안에, 변호사 없이도 계약 리스크를 잡아내는 AI 계약서 리뷰어"**

### 핵심 차별점

1. **가격 공백 공략** — 엔터프라이즈 CLM($15K+/년)과 무료 AI 도구 사이의 $100~$999/월 구간에 최적화
2. **SMB 맞춤 UX** — 법률 용어를 비즈니스 언어로 번역, 법무팀 없는 조직도 즉시 활용
3. **산업별 리스크 템플릿** — SaaS, 제조, 유통, 부동산 등 산업 특화 조항 분석
4. **한국어 + 영어 네이티브** — 국내 기업의 국문/영문 혼합 계약서 동시 지원

## 2. 타겟 사용자 & 유저 저니

### 2.1 타겟 사용자 페르소나

#### Persona A: 스타트업 COO "민지" (Primary)

| 항목 | 상세 |
|------|------|
| **역할** | 시리즈 A 스타트업 COO / 경영지원 총괄 |
| **회사 규모** | 20~50명 |
| **법무 인력** | 없음 (외부 로펌 스팟 의뢰) |
| **월 계약 건수** | 5~15건 (SaaS 구독, 파트너십, NDA, 용역) |
| **핵심 Pain** | 계약서 리뷰 요청 시 로펌 응답까지 3~5일 소요, 건당 50~200만원 |
| **핵심 Goal** | 명백한 리스크 조항을 사전에 걸러내고, 정말 필요한 건만 변호사에게 의뢰 |
| **기술 숙련도** | Google Workspace / Notion 능숙, 법률 지식은 기초 수준 |

#### Persona B: 중견기업 법무 담당자 "승환" (Secondary)

| 항목 | 상세 |
|------|------|
| **역할** | 매출 500억 중견기업 법무팀 사원 (팀 3명) |
| **회사 규모** | 200~500명 |
| **월 계약 건수** | 30~80건 |
| **핵심 Pain** | 반복적인 NDA/기본계약서 검토에 시간 소모, 핵심 거래 검토에 집중 불가 |
| **핵심 Goal** | 정형 계약은 AI로 자동 처리하고, 비정형/고위험 건에 리소스 집중 |
| **기술 숙련도** | CLM 도구 경험 있음, 법률 전문가 |

#### Persona C: 프리랜서/소상공인 "지수" (Tertiary)

| 항목 | 상세 |
|------|------|
| **역할** | IT 프리랜서 또는 1인 사업자 |
| **월 계약 건수** | 1~5건 |
| **핵심 Pain** | 계약서를 "읽긴 읽지만" 리스크 조항을 식별할 역량 부족 |
| **핵심 Goal** | 불리한 조항을 사전에 인지하고 협상 포인트 확보 |

## 3. 유저 스토리 (10개)

### P0 — Must Ship (MVP 필수)

#### US-01: 계약서 업로드 및 AI 분석

| 항목 | 내용 |
|------|------|
| **As a** | 스타트업 COO (민지) |
| **I want to** | PDF/DOCX 계약서를 업로드하면 AI가 자동으로 분석해주길 |
| **So that** | 변호사 없이도 5분 내에 계약서의 핵심 리스크를 파악할 수 있다 |

**Acceptance Criteria:**
- [ ] PDF, DOCX, HWP 형식 지원 (최대 50페이지, 20MB)
- [ ] 업로드 후 5분 이내에 분석 완료 (95th percentile)
- [ ] 분석 진행률 실시간 표시 (업로드 → OCR → 조항 파싱 → 리스크 분석 → 완료)
- [ ] 한국어, 영어 계약서 모두 지원
- [ ] 분석 실패 시 사유와 함께 에러 메시지 표시

---

#### US-02: 리스크 리포트 조회

| 항목 | 내용 |
|------|------|
| **As a** | 스타트업 COO (민지) |
| **I want to** | 분석 결과를 리스크 수준별로 정리된 리포트로 보고 싶다 |
| **So that** | 어떤 조항이 위험한지, 왜 위험한지 비법률 용어로 이해할 수 있다 |

**Acceptance Criteria:**
- [ ] 리스크 등급 3단계 표시: 🔴 High / 🟡 Medium / 🟢 Low
- [ ] 각 리스크 항목에: 원문 조항 하이라이트 + 평문 설명 + 위험 사유
- [ ] 전체 리스크 스코어 (0~100) 대시보드 상단에 표시
- [ ] 리스크 항목 클릭 시 원문 해당 위치로 스크롤 이동
- [ ] 리포트 PDF 다운로드 가능

#### US-03: 수정 제안 (Redline Suggestion)

| 항목 | 내용 |
|------|------|
| **As a** | 스타트업 COO (민지) |
| **I want to** | 위험 조항에 대해 AI가 대체 문구를 제안해주길 |
| **So that** | 상대방에게 바로 수정 요청을 보낼 수 있다 |

**Acceptance Criteria:**
- [ ] High/Medium 리스크 조항 각각에 1~2개의 대안 문구 제시

- [ ] 수정 제안 선택 시 원문에 레드라인 마크업 자동 적용
- [ ] 레드라인 적용된 DOCX 파일 내보내기 지원
- [ ] 수정 제안에 "이 수정이 필요한 이유" 한 줄 설명 포함

## 3. 비기능 요구사항

### 3.1 성능 (Performance)

| 항목 | 요구사항 | 근거 |
|------|---------|------|
| **계약서 분석 완료 시간** | 10페이지 이하 문서 **≤ 3분**, 50페이지 이하 **≤ 5분** | Part 1 핵심 가치 제안 "5분 이내 자동 분석 완료" |
| **업로드 응답 시간** | 파일 업로드 후 파싱 시작까지 **≤ 2초** | 첫 리뷰 경험에서 즉각 피드백 필요 |

| **동시 분석 처리** | 팀 플랜 기준 **동시 10건** 분석 큐 처리 | Persona B "월 30~80건" 워크로드 대응 |
| **파일 크기 제한** | 단일 파일 **≤ 50MB**, PDF/DOCX/HWP 지원 | 국내 기업 한글(HWP) 계약서 필수 지원 |
| **API 가용성** | **99.5% uptime** (월간), 계획 유지보수 제외 | SMB 신뢰 기준선 |
| **리포트 생성** | 리스크 리포트 PDF 내보내기 **≤ 10초** | 공유/보관 워크플로우 지원 |

### 3.2 보안 (Security)

| 항목 | 요구사항 | 상세 |
|------|---------|------|
| **데이터 암호화** | 전송 중 TLS 1.3, 저장 시 AES-256 | 계약서는 기밀 문서 — 암호화 필수 |
| **파일 보존 정책** | 분석 완료 후 원본 파일 **90일 보존**, 이후 자동 삭제. 사용자 수동 즉시 삭제 가능 | GDPR/개인정보보호법 대응 |
| **LLM 데이터 격리** | 업로드된 계약서 데이터가 LLM 학습에 **절대 사용되지 않음** 보장. 전용 API 엔드포인트 또는 자체 모델 사용 | 고객 기밀 유지 핵심 요건 |
| **인증** | Google SSO, 이메일+비밀번호 (bcrypt), 선택적 MFA(TOTP) | Part 1 유저 저니 "구글 SSO 30초 가입" 연계 |
| **권한 관리 (RBAC)** | 역할: `Owner` · `Admin` · `Member` · `Viewer` | 팀 플랜 멀티유저 접근 제어 |
| **감사 로그** | 업로드/분석/내보내기/삭제 등 모든 핵심 행위 기록, **1년 보존** | 컴플라이언스 및 내부 감사 대응 |
| **SOC 2 Type II** | MVP 이후 6개월 내 인증 착수 | 중견기업(Persona B) 도입 의사결정의 게이트 요건 |

### 3.3 접근성 (Accessibility)

### 5.2 엔티티 상세

#### Organization (조직)

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | UUID (PK) | 조직 고유 ID |
| `name` | VARCHAR(200) | 조직명 |
| `industry` | ENUM | 업종 (saas, manufacturing, realestate, service, other) |
| `plan` | ENUM | 구독 플랜 (free, starter, professional, team) |
| `created_at` | TIMESTAMP | 생성일 |
| `updated_at` | TIMESTAMP | 수정일 |

#### User (사용자)

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | UUID (PK) | 사용자 고유 ID |
| `email` | VARCHAR(320) UNIQUE | 이메일 (로그인 식별자) |
| `password_hash` | VARCHAR(256) NULL | bcrypt 해시 (SSO 시 NULL) |
| `name` | VARCHAR(100) | 이름 |
| `auth_provider` | ENUM | 인증 방식 (email, google) |
| `role_preference` | ENUM | 역할 (executive, legal, sales, freelancer) |
| `mfa_enabled` | BOOLEAN | MFA 활성 여부 (기본: false) |
| `mfa_secret` | VARCHAR(256) NULL | TOTP 시크릿 (암호화 저장) |
| `language` | ENUM | 선호 언어 (ko, en) |
| `created_at` | TIMESTAMP | 가입일 |
| `last_login_at` | TIMESTAMP | 최근 로그인 |

#### OrgMember (조직-사용자 관계)

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | UUID (PK) | |
| `org_id` | UUID (FK → Organization) | 조직 |
| `user_id` | UUID (FK → User) | 사용자 |
| `role` | ENUM | 권한 (owner, admin, member, viewer) |
| `invited_at` | TIMESTAMP | 초대일 |
| `joined_at` | TIMESTAMP NULL | 수락일 |
| `status` | ENUM | 상태 (active, pending, deactivated) |

#### Contract (계약서)

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | UUID (PK) | 계약서 고유 ID |

## 6. 성공 지표 (KPI)

### 6.1 핵심 KPI 5개

| # | KPI | 정의 | 목표값 (출시 6개월) | 측정 방법 |
|---|-----|------|-------------------|-----------|
| **K1** | **Activation Rate** | 가입 후 7일 내 첫 계약서 분석 완료 비율 | **≥ 60%** | `COUNT(첫 분석 완료 within 7d) / COUNT(신규 가입)` |
| **K2** | **Weekly Active Reviewer** | 주 1회 이상 계약서 분석을 수행한 유저 수 | **≥ 500 WAR** (6개월차) | 주간 unique user with `Analysis.status = completed` |
| **K3** | **분석 완료율** | 업로드된 계약서 중 분석이 성공적으로 완료된 비율 | **≥ 95%** | `COUNT(completed) / COUNT(uploaded)` |
| **K4** | **MRR (Monthly Recurring Revenue)** | 월간 반복 매출 | **≥ $30,000** (6개월차) | Stripe MRR 집계 |
| **K5** | **NPS (Net Promoter Score)** | 순추천지수 | **≥ 40** | 분기별 인앱 서베이 (리포트 10건 이상 사용자 대상) |

### 6.2 보조 지표

| 지표 | 목표값 | 비고 |
|------|--------|------|
| 평균 분석 소요 시간 | ≤ 3분 (10p 기준) | 성능 요구사항 검증 |
| 수정 제안 채택률 | ≥ 30% | `Suggestion.accepted = true` 비율 |
| Free → Paid 전환율 | ≥ 8% | 무료 3건 소진 후 30일 내 유료 전환 |
| 팀 플랜 확장률 | ≥ 15% | Starter/Pro → Team 업그레이드 비율 (12개월) |
| 리포트 공유율 | ≥ 20% | 리포트 생성 건 중 ShareLink 생성 비율 |
| Churn Rate (월간) | ≤ 5% | 유료 구독 해지율 |

### 6.3 KPI-기능 매핑

```
K1 Activation Rate
 └─ P3 온보딩 위저드 (업종/유형 사전 설정 → 첫 분석 마찰 감소)
 └─ P5 업로드 UX (드래그&드롭 + 자동감지 → 설정 부담 최소화)
 └─ P6 분석 진행 (실시간 진행률 → 이탈 방지)

K2 Weekly Active Reviewer
 └─ P4 대시보드 (요약 카드 + 인사이트 → 재방문 유도)
 └─ P14 위클리 다이제스트 (이메일 리인게이지먼트)
 └─ P10 보관함 (이력 축적 → 습관 형성)

K3 분석 완료율
 └─ HWP/DOCX/PDF 파서 안정성
 └─ 비동기 분석 큐 + 재시도 로직
 └─ P6 실패 시 사용자 안내 + 재시도 버튼

K4 MRR
 └─ P1 랜딩 요금제 노출 (가격 공백 포지셔닝)
 └─ P13 빌링 (셀프서비스 업그레이드)
 └─ P12 팀 관리 (시트 추가 → ARPU 증가)

K5 NPS
 └─ P7/P8 리포트 품질 (비즈니스 언어 번역 + 협상 팁)
 └─ P9 변호사 연결 (고위험 시 명확한 다음 단계 제시)
 └─ P11 커스텀 템플릿 (조직 맞춤 → 정확도 향상)
```
