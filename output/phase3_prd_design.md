## 3. 비기능 요구사항

### 3.1 성능 (Performance)

| 항목 | 요구사항 | 근거 |
|------|---------|------|
| **계약서 분석 완료 시간** | 10페이지 이하 문서 **≤ 3분**, 50페이지 이하 **≤ 5분** | Part 1 핵심 가치 제안 "5분 이내 자동 분석 완료" |
| **업로드 응답 시간** | 파일 업로드 후 파싱 시작까지 **≤ 2초** | 첫 리뷰 경험에서 즉각 피드백 필요 |
| **대시보드 로딩** | 초기 로드 **≤ 1.5초**, 이후 내비게이션 **≤ 500ms** | SPA 기준 Core Web Vitals LCP 목표 |
| **동시 분석 처리** | 팀 플랜 기준 **동시 10건** 분석 큐 처리 | Persona B "월 30~80건" 워크로드 대응 |
| **파일 크기 제한** | 단일 파일 **≤ 50MB**, PDF/DOCX/HWP 지원 | 국내 기업 한글(HWP) 계약서 필수 지원 |
| **API 가용성** | **99.5% uptime** (월간), 계획 유지보수 제외 | SMB 신뢰 기준선 |
| **리포트 생성** | 리스크 리포트 PDF 내보내기 **≤ 10초** | 공유/보관 워크플로우 지원 |

### 3.2 보안 (Security)

| 항목 | 요구사항 | 상세 |
|------|---------|------|
| **데이터 암호화** | 전송 중 TLS 1.3, 저장 시 AES-256 | 계약서는 기밀 문서 — 암호화 필수 |
| **파일 보존 정책** | 분석 완료 후 원본 파일 **90일 보존**, 이후 자동 삭제. 사용자 수동 즉시 삭제 가능 | GDPR/개인정보보호법 대응 |
| **LLM 데이터 격리** | 업로드된 계약서 데이터가 LLM 학습에 **절대 사용되지 않음** 보장. 전용 API 엔드포인트 또는 자체 모델 사용 | 고객 기밀 유지 핵심 요건 |
| **인증** | Google SSO, 이메일+비밀번호 (bcrypt), 선택적 MFA(TOTP) | Part 1 유저 저니 "구글 SSO 30초 가입" 연계 |
| **권한 관리 (RBAC)** | 역할: `Owner` · `Admin` · `Member` · `Viewer` | 팀 플랜 멀티유저 접근 제어 |
| **감사 로그** | 업로드/분석/내보내기/삭제 등 모든 핵심 행위 기록, **1년 보존** | 컴플라이언스 및 내부 감사 대응 |
| **SOC 2 Type II** | MVP 이후 6개월 내 인증 착수 | 중견기업(Persona B) 도입 의사결정의 게이트 요건 |

### 3.3 접근성 (Accessibility)

| 항목 | 요구사항 |
|------|---------|
| **WCAG 준수** | 2.1 AA 수준 |
| **키보드 네비게이션** | 모든 핵심 플로우(업로드 → 리포트 확인 → 내보내기) 키보드만으로 완료 가능 |
| **스크린 리더** | 리스크 리포트의 심각도 라벨/점수를 aria-label로 제공 |
| **색상 대비** | 리스크 레벨 표시 시 색상 + 아이콘 + 텍스트 라벨 병용 (색맹 대응) |
| **반응형 디자인** | 데스크톱(1280px+) 최적화, 태블릿(768px+) 지원. 모바일은 리포트 열람만 지원 |
| **다국어** | 한국어(기본), 영어. UI 및 리포트 출력 언어 독립 선택 가능 |

### 3.4 확장성 & 운영

| 항목 | 요구사항 |
|------|---------|
| **인프라** | 컨테이너 기반 (Kubernetes), auto-scaling |
| **분석 큐** | 비동기 작업 큐 (메시지 브로커) — 트래픽 급증 시 graceful degradation |
| **모니터링** | APM + 에러 트래킹 + 분석 파이프라인별 레이턴시 대시보드 |
| **백업** | DB 일일 자동 백업, 30일 보존, 크로스리전 복제 |

## 6. 성공 지표 (KPI)

### 6.1 핵심 KPI 5개

| # | KPI | 정의 | 목표값 (출시 6개월) | 측정 방법 |
|---|-----|------|-------------------|-----------|
| **K1** | **Activation Rate** | 가입 후 7일 내 첫 계약서 분석 완료 비율 | **≥ 60%** | `COUNT(첫 분석 완료 within 7d) / COUNT(신규 가입)` |
| **K2** | **Weekly Active Reviewer** | 주 1회 이상 계약서 분석을 수행한 유저 수 | **≥ 500 WAR** (6개월차) | 주간 unique user with `Analysis.status = completed` |
| **K3** | **분석 완료율** | 업로드된 계약서 중 분석이 성공적으로 완료된 비율 | **≥ 95%** | `COUNT(completed) / COUNT(uploaded)` |
| **K4** | **MRR (Monthly Recurring Revenue)** | 월간 반복 매출 | **≥ $30,000** (6개월차) | Stripe MRR 집계 |
| **K5** | **NPS (Net Promoter Score)** | 순추천지수 | **≥ 40** | 분기별 인앱 서베이 (리포트 10건 이상 사용자 대상) |

### 6.2 보조 지표

| 지표 | 목표값 | 비고 |
|------|--------|------|
| 평균 분석 소요 시간 | ≤ 3분 (10p 기준) | 성능 요구사항 검증 |
| 수정 제안 채택률 | ≥ 30% | `Suggestion.accepted = true` 비율 |
| Free → Paid 전환율 | ≥ 8% | 무료 3건 소진 후 30일 내 유료 전환 |
| 팀 플랜 확장률 | ≥ 15% | Starter/Pro → Team 업그레이드 비율 (12개월) |
| 리포트 공유율 | ≥ 20% | 리포트 생성 건 중 ShareLink 생성 비율 |
| Churn Rate (월간) | ≤ 5% | 유료 구독 해지율 |

### 6.3 KPI-기능 매핑

```
K1 Activation Rate
 └─ P3 온보딩 위저드 (업종/유형 사전 설정 → 첫 분석 마찰 감소)
 └─ P5 업로드 UX (드래그&드롭 + 자동감지 → 설정 부담 최소화)
 └─ P6 분석 진행 (실시간 진행률 → 이탈 방지)

K2 Weekly Active Reviewer
 └─ P4 대시보드 (요약 카드 + 인사이트 → 재방문 유도)
 └─ P14 위클리 다이제스트 (이메일 리인게이지먼트)
 └─ P10 보관함 (이력 축적 → 습관 형성)

K3 분석 완료율
 └─ HWP/DOCX/PDF 파서 안정성
 └─ 비동기 분석 큐 + 재시도 로직
 └─ P6 실패 시 사용자 안내 + 재시도 버튼

K4 MRR
 └─ P1 랜딩 요금제 노출 (가격 공백 포지셔닝)
 └─ P13 빌링 (셀프서비스 업그레이드)
 └─ P12 팀 관리 (시트 추가 → ARPU 증가)

K5 NPS
 └─ P7/P8 리포트 품질 (비즈니스 언어 번역 + 협상 팁)
 └─ P9 변호사 연결 (고위험 시 명확한 다음 단계 제시)
 └─ P11 커스텀 템플릿 (조직 맞춤 → 정확도 향상)
```
