## 2. 기술 블로그 포스트

### 제목

**"AI 계약서 리뷰 SaaS를 8주 만에 만든 이야기 — ClauseGuard의 기술 스택, 아키텍처, 그리고 삽질 기록"**

### 목차 & 각 섹션 요약

**1. 왜 만들었나: $500 변호사 비용과 5분 AI의 간극**
- 스타트업 운영 중 계약서 리뷰의 현실적 고통 (건당 $500~$5,000, 3~7일 소요)
- Legal AI 시장 규모: 2025년 $31~40억 → 2030년 $108억 (CAGR 28.3%)
- AI 계약서 리뷰 세그먼트가 Legal AI 내 최고 성장률(CAGR 31.8%)
- $100~$500/월 가격대의 명확한 시장 공백 발견

**2. 경쟁사 분석에서 찾은 기회: 9가지 공통 불만**
- Ironclad($32억 유니콘), SpotDraft, Juro 등 주요 경쟁사 5개 분석
- G2/Capterra/Reddit에서 추출한 SMB 사용자 9대 불만: 가격 불투명, 과잉 기능, 구현 기간 4~6개월, 숨겨진 비용(TCO 1.5~2배), 강제 장기 계약 등
- 결론: "법무팀 없는 SMB가 5분 만에 쓸 수 있는 도구"가 존재하지 않았다

**3. 기술 스택 선정 — 왜 Next.js + FastAPI + Claude API인가**
- 프론트엔드: Next.js 15 + TypeScript + Tailwind CSS → SSR로 SEO 확보, 서버 컴포넌트로 PDF 렌더링
- 백엔드: FastAPI (Python) → LangChain/LlamaIndex 생태계, async LLM 스트리밍
- DB: PostgreSQL(Supabase) + pgvector + Redis → 임베딩 검색 + 캐싱
- 배포: Vercel(프론트) + Railway(백엔드) → 초기 비용 최소화
- Claude API 단일 의존 리스크 → LiteLLM 멀티 LLM 추상화 레이어로 해결

**4. 핵심 파이프라인: 업로드부터 리포트까지 5분의 여정**
- Step 1: 업로드 → S3 저장 + 파싱 워커 실행
- Step 2: pdfplumber/python-docx로 조항 단위 청킹
- Step 3: 임베딩 생성 → pgvector 저장 → 유사 조항 검색
- Step 4: Claude API 병렬 분석 (asyncio) → 조항별 리스크 분류
- Step 5: 결과 집계 → SSE로 실시간 진행률 → 리포트 생성
- 아키텍처 다이어그램과 각 레이어의 역할 상세 설명

**5. 가장 힘들었던 3가지 기술 문제와 해결법**
- HWP/스캔 PDF 파싱 지옥: pdfplumber 실패 시 Tesseract OCR 폴백, 파싱 신뢰도 점수 시스템
- LLM 분석 일관성: 동일 조항에 다른 결과 → Few-shot 100개+ 프롬프트 고정 + 다수결 재분석
- API 비용 폭발: 100페이지 계약서 → 조항 해시 기반 Redis 캐싱 + Haiku 1차 필터링 → Sonnet High 리스크만 재분석

**6. 법률 용어를 비즈니스 언어로: 프롬프트 엔지니어링 비하인드**
- "손해배상 책임 제한"을 "이 조항 때문에 문제가 생겨도 상대방은 최대 100만원만 물어줍니다"로 번역하는 과정
- 한국어-영어 교차 검토: 번역 과정에서 발생하는 의미 차이 감지
- 산업별(SaaS, 제조, 부동산) 리스크 체크리스트 설계 과정

**7. 수익 모델과 가격 설정의 논리**
- Starter $99 / Pro $299 / Business $499 — 가격 공백 정조준
- 변호사 대비 ROI: 스타트업 기준 97~99% 비용 절감
- 마진 구조: AI API 비용 포함 80%+ 총이익률
- PLG + SEO 기반 CAC $800 목표

**8. 출시 후 첫 30일: 숫자로 보는 초기 트랙션**
- MVP 8주 개발 타임라인 회고
- 첫 30일 목표 KPI: 가입 전환율 ≥8%, 활성화율 ≥60%, 무료→유료 전환율 ≥15%
- 로드맵: 계약 유형 자동 분류, 버전 비교(Diff), 커스텀 리스크 룰, API 제공

저는 스타트업을 운영하는 대표입니다.

매달 5~15건의 계약서가 쌓이는데, 매번 변호사한테 맡기면 건당 50~200만원($400~$1,600)에 3~5일이 걸렸습니다. 그렇다고 안 맡기면? 자동갱신 조항을 놓쳐서 1년치 구독료를 날린 적도 있고, 경쟁 제한 조항이 있는 줄도 모르고 서명한 적도 있습니다.

**문제:** 엔터프라이즈 CLM(Ironclad, Juro 등)은 연 $15,000 이상이고 구축에 4~6개월. ChatGPT에 "이 계약서 봐줘"라고 던지기에는 정확도와 일관성이 부족. 그 사이에 $100~$500/월짜리 전문 도구가 없었습니다.

**해결:** ClauseGuard를 만들었습니다.

**작동 방식:**
1. PDF/DOCX 계약서 업로드
2. AI가 조항 단위로 분리해서 각각 분석
3. 5분 내에 리스크 리포트 제공:
   - 조항별 위험도 점수(1-100) + 🔴🟡🟢
   - "왜 위험한지" 비법률인도 이해할 수 있는 설명
   - 균형 잡힌 수정안 자동 생성
4. 레드라인 DOCX 내보내기 → 바로 상대방에게 전달

**비교표:**

|---|---|---|---|---|
| 비용 | $400~$5,000/건 | $15K+/년 | ~무료 | **$99~$499/월** |
| 속도 | 3~7일 | 즉시 (구축 후) | 즉시 | **5분** |
